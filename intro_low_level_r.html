R objects

R has variables that are used to access objects:

  • functions
  • data

Functions

A function provides an action over objects or R itself.

If f is a function, f(...) applies the function to its parameters while f is the function as an object.

Data

Data objects can be separated into two main categories:

  • vector: a finite set of values of a given type (like integers)
  • list: a finite set of vectors (possibly of different types)

Numerical vectors

x <- 2
x
[1] 2
## x is a numeric vector
class(x)
[1] "numeric"
## the type of x content is double
typeof(x)
[1] "double"
y <- 2L ## make clear to R that 2 is an integer (add L)
y
[1] 2
## y is a integer vector
class(y)
[1] "integer"
## the type of y content is integer
typeof(y)
[1] "integer"
## square root calculation
z <- sqrt(y)
class(z)
[1] "numeric"
typeof(z)
[1] "double"
z
[1] 1.414214
## c is a construction function
x2 <- c(-2, 3, -1.5)
class(x2)
[1] "numeric"
typeof(x2)
[1] "double"
x2
[1] -2.0  3.0 -1.5

Most of the functions operate in vectorized mode, i.e. their actions are applied to each element of a vector independently from the other elements.

## absolute value
y2 <- abs(x2)
y2
[1] 2.0 3.0 1.5
sqrt(abs(x2))
[1] 1.414214 1.732051 1.224745
sqrt(x2)
[1]      NaN 1.732051      NaN

The special value NaN stands for Not a Number and is used when a numerical value cannot be associated to a mathematical expression.

Individual access

Logical

The logical type is obtained as the result of logical operations such as comparisons (comparisons are vectorized).

z <- seq(-2, 2)
y <- z > 0
y
[1] FALSE FALSE FALSE  TRUE  TRUE
class(y)
[1] "logical"
typeof(y)
[1] "logical"
z == 0 ## equality
[1] FALSE FALSE  TRUE FALSE FALSE
z != 0 ## difference
[1]  TRUE  TRUE FALSE  TRUE  TRUE
z >= 1
[1] FALSE FALSE FALSE  TRUE  TRUE
y <- c(TRUE, FALSE, FALSE, TRUE)
y
[1]  TRUE FALSE FALSE  TRUE
## useless
y == TRUE
[1]  TRUE FALSE FALSE  TRUE
## non explicit
y != TRUE
[1] FALSE  TRUE  TRUE FALSE
## negation operator
!y
[1] FALSE  TRUE  TRUE FALSE

In addition to the negation operator, we have or | and and &.

z <- seq(-5, 5)
(z >= 4) | (z <= -3)
 [1]  TRUE  TRUE  TRUE FALSE FALSE FALSE FALSE FALSE FALSE  TRUE  TRUE
(z < 3) & (z >= -2)
 [1] FALSE FALSE FALSE  TRUE  TRUE  TRUE  TRUE  TRUE FALSE FALSE FALSE
(z[2] > 4) | (z < 0) ## values are reused
 [1]  TRUE  TRUE  TRUE  TRUE  TRUE FALSE FALSE FALSE FALSE FALSE FALSE

List

A list is a collection of vectors that may have different types.

Basic operations

x <- list(c(1, 2, 3), c("x", "y"), TRUE)
x
[[1]]
[1] 1 2 3

[[2]]
[1] "x" "y"

[[3]]
[1] TRUE
class(x)
[1] "list"
typeof(x)
[1] "list"
length(x)
[1] 3
lengths(x) ## length applied to each element of the list
[1] 3 2 1

Access to elements of the list is different should not be confused to sublist extraction.

x[1] ## sublist
[[1]]
[1] 1 2 3
x[[1]] ## element
[1] 1 2 3
class(x[1])
[1] "list"
length(x[1])
[1] 1
class(x[[1]])
[1] "numeric"
length(x[[1]])
[1] 3
x[-1]
[[1]]
[1] "x" "y"

[[2]]
[1] TRUE
